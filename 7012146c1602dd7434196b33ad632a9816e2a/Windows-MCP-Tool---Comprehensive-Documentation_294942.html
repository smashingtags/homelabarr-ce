Windows MCP Tool - Comprehensive Documentation

Overview

The Windows MCP (Model Context Protocol) Tool is a powerful automation framework that enables programmatic interaction with Windows desktop applications, system components, and UI elements. It provides a comprehensive set of tools for desktop automation, testing, and productivity enhancement.

Getting Started

Prerequisites

  • Windows 10/11 operating system

  • MCP-compatible client (Claude Desktop, etc.)

  • Administrator privileges may be required for certain operations

Installation

The Windows MCP Tool is typically installed as part of an MCP client package. Ensure your MCP client has the Windows MCP server configured and running.

Basic Concepts

Coordinates System: All UI interactions use screen coordinates (x, y) where (0,0) is the top-left corner of the primary display.

Element Detection: The State Tool provides lists of interactive, informative, and scrollable elements with their coordinates and properties.

Application Context: Tools can interact with any visible application window or system component.

Core Tools Reference

1. State Tool

Function: Windows-MCP:State-Tool

Captures comprehensive desktop state including focused applications, opened applications, and interactive UI elements.

Parameters:

  • use_vision (boolean, default: false): Include visual screenshot when true

Returns:

  • Focused application information

  • List of opened applications with status and dimensions

  • Interactive elements (buttons, text fields, menus)

  • Informative content (labels, text, status indicators)

  • Scrollable areas

Example Usage:

# Get current desktop state
state = windows_mcp.state_tool(use_vision=False)

2. Click Tool

Function: Windows-MCP:Click-Tool

Performs mouse click operations at specified coordinates.

Parameters:

  • loc (array): [x, y] coordinates for click location

  • button (string): Mouse button - "left", "right", or "middle" (default: "left")

  • clicks (integer): Number of clicks - 1 (single), 2 (double), 3 (triple) (default: 1)

Example Usage:

# Single left-click at coordinates
windows_mcp.click_tool(loc=[100, 200])

# Double-click
windows_mcp.click_tool(loc=[100, 200], clicks=2)

# Right-click for context menu
windows_mcp.click_tool(loc=[100, 200], button="right")

3. Type Tool

Function: Windows-MCP:Type-Tool

Types text into input fields or focused elements.

Parameters:

  • loc (array): [x, y] coordinates of target element

  • text (string): Text to type

  • clear (boolean, default: false): Clear existing text before typing

Example Usage:

# Type into a text field
windows_mcp.type_tool(loc=[300, 150], text="Hello World")

# Clear field and type new text
windows_mcp.type_tool(loc=[300, 150], text="New Text", clear=True)

4. Scroll Tool

Function: Windows-MCP:Scroll-Tool

Scrolls content at specified coordinates or current mouse position.

Parameters:

  • loc (array, optional): [x, y] coordinates for scroll location

  • direction (string): "up", "down", "left", or "right" (default: "down")

  • type (string): "vertical" or "horizontal" (default: "vertical")

  • wheel_times (integer): Number of scroll wheel turns (default: 1)

Example Usage:

# Scroll down at specific location
windows_mcp.scroll_tool(loc=[500, 300], direction="down", wheel_times=3)

# Horizontal scroll
windows_mcp.scroll_tool(loc=[500, 300], type="horizontal", direction="right")

5. Launch Tool

Function: Windows-MCP:Launch-Tool

Launches applications from the Windows Start Menu.

Parameters:

  • name (string): Application name as it appears in Start Menu

Example Usage:

# Launch applications
windows_mcp.launch_tool(name="notepad")
windows_mcp.launch_tool(name="calculator")
windows_mcp.launch_tool(name="chrome")

6. Switch Tool

Function: Windows-MCP:Switch-Tool

Switches to a specific application window and brings it to foreground.

Parameters:

  • name (string): Application name to switch to

Example Usage:

# Switch between applications
windows_mcp.switch_tool(name="notepad")
windows_mcp.switch_tool(name="chrome")

7. Shortcut Tool

Function: Windows-MCP:Shortcut-Tool

Executes keyboard shortcuts using key combinations.

Parameters:

  • shortcut (array): List of keys to press simultaneously

Common Shortcuts:

  • Copy: ["ctrl", "c"]

  • Paste: ["ctrl", "v"]

  • Alt+Tab: ["alt", "tab"]

  • Windows+R: ["win", "r"]

  • Print Screen: ["print_screen"]

Example Usage:

# Common shortcuts
windows_mcp.shortcut_tool(shortcut=["ctrl", "c"])  # Copy
windows_mcp.shortcut_tool(shortcut=["ctrl", "v"])  # Paste
windows_mcp.shortcut_tool(shortcut=["alt", "tab"])  # App switcher
windows_mcp.shortcut_tool(shortcut=["win", "r"])   # Run dialog

8. Key Tool

Function: Windows-MCP:Key-Tool

Presses individual keyboard keys including special keys.

Parameters:

  • key (string): Key name to press

Special Keys:

  • Navigation: "up", "down", "left", "right"

  • Action: "enter", "escape", "tab", "space"

  • Editing: "backspace", "delete", "home", "end"

  • Function: "f1" through "f12"

Example Usage:

# Navigation and special keys
windows_mcp.key_tool(key="enter")
windows_mcp.key_tool(key="escape")
windows_mcp.key_tool(key="tab")
windows_mcp.key_tool(key="f5")  # Refresh

9. Drag Tool

Function: Windows-MCP:Drag-Tool

Performs drag and drop operations between two coordinates.

Parameters:

  • from_loc (array): [x, y] starting coordinates

  • to_loc (array): [x, y] destination coordinates

Example Usage:

# Drag from one location to another
windows_mcp.drag_tool(from_loc=[100, 100], to_loc=[300, 300])

10. Move Tool

Function: Windows-MCP:Move-Tool

Moves mouse cursor to specific coordinates without clicking.

Parameters:

  • to_loc (array): [x, y] destination coordinates

Example Usage:

# Move cursor for hover effects
windows_mcp.move_tool(to_loc=[250, 150])

11. Clipboard Tool

Function: Windows-MCP:Clipboard-Tool

Manages clipboard operations for copying and pasting text.

Parameters:

  • mode (string): "copy" or "paste"

  • text (string, optional): Text to copy when mode is "copy"

Example Usage:

# Copy text to clipboard
windows_mcp.clipboard_tool(mode="copy", text="Hello World")

# Get clipboard content
clipboard_content = windows_mcp.clipboard_tool(mode="paste")

12. Wait Tool

Function: Windows-MCP:Wait-Tool

Pauses execution for specified duration.

Parameters:

  • duration (integer): Wait time in seconds

Example Usage:

# Wait for application to load
windows_mcp.wait_tool(duration=3)

13. PowerShell Tool

Function: Windows-MCP:Powershell-Tool

Executes PowerShell commands and returns output.

Parameters:

  • command (string): PowerShell command to execute

Example Usage:

# System information
result = windows_mcp.powershell_tool(command="Get-Process | Select-Object Name, CPU")

# File operations
windows_mcp.powershell_tool(command="Get-ChildItem C:\\ -Directory")

14. Scrape Tool

Function: Windows-MCP:Scrape-Tool

Fetches and converts webpage content to markdown format.

Parameters:

  • url (string): Full URL including protocol (http/https)

Example Usage:

# Scrape webpage content
content = windows_mcp.scrape_tool(url="https://example.com")

Common Use Cases

1. Application Automation

Opening and Configuring Applications:

# Launch and setup development environment
windows_mcp.launch_tool(name="code")
windows_mcp.wait_tool(duration=3)
windows_mcp.shortcut_tool(shortcut=["ctrl", "shift", "p"])  # Command palette
windows_mcp.type_tool(loc=[960, 100], text="Python: Select Interpreter")

Form Filling Automation:

# Automate form completion
windows_mcp.click_tool(loc=[300, 200])  # Click first name field
windows_mcp.type_tool(loc=[300, 200], text="John")
windows_mcp.key_tool(key="tab")  # Move to next field
windows_mcp.type_tool(loc=[300, 250], text="Doe")

2. File Management

Batch File Operations:

# Organize downloads folder
windows_mcp.launch_tool(name="explorer")
windows_mcp.wait_tool(duration=2)
windows_mcp.shortcut_tool(shortcut=["ctrl", "l"])  # Address bar
windows_mcp.type_tool(loc=[500, 50], text="C:\\Users\\%USERNAME%\\Downloads")
windows_mcp.key_tool(key="enter")

3. Testing and Quality Assurance

UI Testing Workflows:

# Test login functionality
windows_mcp.click_tool(loc=[400, 300])  # Username field
windows_mcp.type_tool(loc=[400, 300], text="testuser")
windows_mcp.click_tool(loc=[400, 350])  # Password field
windows_mcp.type_tool(loc=[400, 350], text="password123")
windows_mcp.click_tool(loc=[400, 400])  # Login button
windows_mcp.wait_tool(duration=2)
# Verify successful login by checking for expected elements

4. Data Entry and Processing

Automated Data Input:

# Process spreadsheet data
data_rows = [
    ["John", "Doe", "john@example.com"],
    ["Jane", "Smith", "jane@example.com"]
]

for row in data_rows:
    windows_mcp.click_tool(loc=[200, 100])  # First name
    windows_mcp.type_tool(loc=[200, 100], text=row[0])
    windows_mcp.key_tool(key="tab")
    windows_mcp.type_tool(loc=[300, 100], text=row[1])
    windows_mcp.key_tool(key="tab")
    windows_mcp.type_tool(loc=[400, 100], text=row[2])
    windows_mcp.key_tool(key="enter")  # Submit or move to next row

5. System Administration

System Monitoring and Maintenance:

# Check system status
system_info = windows_mcp.powershell_tool(
    command="Get-WmiObject -Class Win32_ComputerSystem | Select-Object Name, Manufacturer, Model"

    command="Get-Service | Where-Object Status -eq 'Running' | Select-Object Name, Status"

Best Practices

1. Error Handling and Reliability

Use Wait Operations:
Always include appropriate wait times after launching applications or triggering actions that require processing time.

windows_mcp.launch_tool(name="application")
windows_mcp.wait_tool(duration=3)  # Allow application to fully load

Verify State Before Actions:
Use the State Tool to verify current desktop state and locate elements before interacting with them.

# Get current state and verify target application is running
state = windows_mcp.state_tool()
# Check if target application is in the opened apps list

2. Coordinate Management

Dynamic Element Location:
Instead of hardcoding coordinates, use the State Tool to dynamically locate elements by their properties.

Relative Positioning:
Consider using relative positioning from known landmarks when possible.

3. Performance Optimization

Minimize State Queries:
Cache state information when possible and only refresh when necessary.

Batch Operations:
Group related operations together to minimize tool switching and improve efficiency.

4. Security Considerations

Sensitive Data Handling:
Be cautious when automating processes involving sensitive information like passwords or personal data.

PowerShell Command Validation:
Always validate PowerShell commands before execution, especially those that modify system settings or files.

Troubleshooting

Common Issues and Solutions

1. Element Not Found

  • Symptom: Clicks or type operations fail to interact with expected elements

  • Solution: Use State Tool to refresh element locations and verify coordinates

2. Application Launch Failures

  • Symptom: Launch Tool cannot find or start specified application

  • Solution: Verify exact application name as it appears in Start Menu

3. Timing Issues

  • Symptom: Operations fail because application is still loading

  • Solution: Increase wait times and add state verification between operations

4. Permission Errors

  • Symptom: PowerShell commands fail with access denied

  • Solution: Run MCP client as administrator or modify commands to use appropriate permissions

5. Coordinate Misalignment

  • Symptom: Clicks occur in wrong locations

  • Solution: Account for DPI scaling and multiple monitor setups

Debugging Tips

1. Use State Tool for Debugging:

# Capture state for troubleshooting
state = windows_mcp.state_tool(use_vision=True)
# Review interactive elements and their coordinates

2. Test Individual Operations:
Break complex workflows into individual steps and test each operation separately.

3. Log Operations:
Maintain logs of successful operations and their parameters for future reference.

Recipe 3: Bulk Email Processing

def process_bulk_emails(email_data):
    # Open email client
    windows_mcp.launch_tool(name="outlook")
    windows_mcp.wait_tool(duration=5)
    
    for email in email_data:
        # Create new email
        windows_mcp.shortcut_tool(shortcut=["ctrl", "n"])
        windows_mcp.wait_tool(duration=1)
        
        # Fill recipient
        windows_mcp.type_tool(loc=[300, 150], text=email["to"])
        windows_mcp.key_tool(key="tab")
        
        # Fill subject
        windows_mcp.type_tool(loc=[300, 180], text=email["subject"])
        windows_mcp.key_tool(key="tab")
        
        # Fill body
        windows_mcp.type_tool(loc=[300, 250], text=email["body"])
        
        # Send email
        windows_mcp.shortcut_tool(shortcut=["ctrl", "enter"])
        windows_mcp.wait_tool(duration=2)

Recipe 4: System Health Check

def system_health_check():
    # Check CPU usage
    cpu_info = windows_mcp.powershell_tool(
        command="Get-WmiObject -Class Win32_Processor | Select-Object LoadPercentage"
    )
    
    # Check memory usage
    memory_info = windows_mcp.powershell_tool(
        command="Get-WmiObject -Class Win32_OperatingSystem | Select-Object TotalVisibleMemorySize, FreePhysicalMemory"
    )
    
    # Check disk space
    disk_info = windows_mcp.powershell_tool(
        command="Get-WmiObject -Class Win32_LogicalDisk | Select-Object DeviceID, Size, FreeSpace"
    )
    
    # Check running processes
    process_info = windows_mcp.powershell_tool(
        command="Get-Process | Sort-Object CPU -Descending | Select-Object -First 10"
    )
    
    return {
        "cpu": cpu_info,
        "memory": memory_info,
        "disk": disk_info,
        "top_processes": process_info
    }

monitor_info = windows_mcp.powershell_tool(
    command="Get-WmiObject -Class Win32_DesktopMonitor | Select-Object Name, ScreenWidth, ScreenHeight"
)

Performance Monitoring

For long-running automation scripts, implement performance monitoring to track execution times and resource usage:

def monitor_performance():

    # ... automation operations ...
    end_time = time.time()
