# MHA Discord Chat - January 2021: The PlexGuide Collapse

## Context
This Discord conversation from January 2021 captures the moment when PlexGuide (PG) suddenly vanished, leaving the community scrambling. The chat reveals the tensions, personalities, and events that led to the creation of PTS (later MHA-Team).

## Key Participants
- **Sammykins** - Owner of the Discord server, took ownership of PTS repository
- **Piltover | Vander/DP** - Long-time community member, helped revive the project
- **halomore** - Michael (current HomelabARR maintainer)
- **Xployt** - SudoBox project founder
- **DrgnFyre** - Community member with concerns about project stability
- **Man1234** - Community observer
- **Doob/MrDoob** - Former PG developer who nuked the wiki and left destructive code

## Timeline of Events

### January 16-17, 2021: The Collapse
- PlexGuide forum and Discord suddenly vanished
- Website gone, only GitHub remained
- Community in shock, no official explanation

### Key Revelations

#### About Admin (PlexGuide Owner)
- Pattern of behavior: "Build community by stealing other people code an renaming it PG"
- Would disappear for months, return, then purge everything
- Multiple data dumps and history erasures
- Only kept project running when receiving donations

#### About Doob/MrDoob
- Skilled coder but destructive when leaving
- Added code to "nuke other peoples server via auto update"
- Destroyed the wiki on his way out
- Coded hard auto-updates that were difficult to remove
- Challenged people to find all hooks he left in the code

#### About Sammykins
- Provided stable home for PTS repository
- Self-described as "a real cunt, the best cunt, 10/10 cunt" (British humor)
- Made herself owner to preserve the code forever
- Never had anything to gain or lose from the project
- Moved on to running Debian and Docker without management tools

#### About SudoBox
- New project attempting to replace PlexGuide
- Doob was coding for them (causing community concern)
- Closed source (raising red flags)
- Xployt leading with promise of better governance

## Community Dynamics

### The Reconstruction Effort
- Community members worked together to piece back the project
- Created PTS as a stable, community-owned alternative
- Focus on preventing another collapse

### Trust Issues
- Doob earned "admin 2.0 rank" for his destructive behavior
- Community wary of any project involving original PG team
- Emphasis on distributed ownership to prevent single point of failure

### Michael's (halomore) Position
- Present during the drama but maintained neutrality
- Acknowledged Doob helped him despite the history
- Advocated for moving forward rather than dwelling on past
- Joined SudoBox Discord to observe

## Lessons Learned

1. **Single ownership is dangerous** - Projects can vanish overnight
2. **Open source matters** - Closed source projects can hide malicious code
3. **Community ownership is key** - Distributed control prevents abuse
4. **Documentation must be preserved** - Wiki destruction hurt many users
5. **Trust is earned slowly, lost quickly** - Destructive exits poison future collaborations

## Impact on HomelabARR

This history directly influenced HomelabARR's design principles:
- Complete transparency and open source
- No single point of failure
- Community-first approach
- Preserved documentation from PTS/MHA
- Focus on stability over constant updates
- No forced auto-updates
- Clear separation from toxic personalities

## The WSL2 Pioneer

Sammykins is notably remembered for:
- Pioneering WSL2 support for the project
- Providing stable Discord community
- Preserving code repository forever
- British humor that lightened tense situations
